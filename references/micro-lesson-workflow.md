# Eddie-style micro-lesson workflow

Use this for LinkedIn posts, newsletters, short essays, X threads, and educational content.

## Structure

```text
small scene or observation
lesson
why it works
application
short close
```

## What makes it work

The reader should feel like they noticed the lesson with you.

Do not open with:

```text
Today I want to talk about conversion copy.
```

Open with a moment:

```text
The best CTA I saw this week was not a button.
```

## Workflow

1. Find a small scene.
2. State what most people miss.
3. Name the lesson.
4. Explain the mechanism in plain language.
5. Show how to use it.
6. End with a line that clicks.

## Micro-lesson templates

### Observation to lesson

```text
I noticed [specific thing].

Most people would call it [surface interpretation].

But it is really [copywriting lesson].

That works because [mechanism].

Use it when [application].

[short close]
```

### Mistake to lesson

```text
I used to write [weak pattern].

It sounded fine.
It did not sell.

The problem was [specific reason].

Now I do [better pattern].

[lesson]
```

### Before and after

```text
Before:
[weak line]

After:
[strong line]

The difference is not style.
It is [mechanism].
```

## Rules

- One lesson per piece.
- Proof or example before abstraction.
- No long preamble.
- No generic motivational ending.
- End when the lesson lands.
