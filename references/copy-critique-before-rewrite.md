# Copy critique before rewrite

Use this when improving existing copy.

## Rule

Do not rewrite from zero until you know what must be preserved.

A bad rewrite can remove the only thing that was working.

## Diagnosis workflow

1. Identify the job of the copy.
2. Identify the reader’s awareness stage.
3. Mark the strongest existing line.
4. Mark any proof, phrase, or claim worth preserving.
5. Identify the conversion leak.
6. Identify the main objection not answered.
7. Rewrite only after diagnosis.

## Output format

```markdown
## Quick diagnosis
- Job of the copy:
- Strongest line to preserve:
- Main leak:
- Missing proof/objection:

## Rewrite
[copy]

## What changed
- [specific change]
- [specific change]
```

## Awareness stages

| Stage | Reader state | Copy job |
|---|---|---|
| Unaware | Does not know the problem | Make the pain visible |
| Problem aware | Feels pain | Name the problem better than they can |
| Solution aware | Knows options | Show why this mechanism is different |
| Product aware | Knows the offer | Prove fit and reduce risk |
| Most aware | Ready | Make action easy |
