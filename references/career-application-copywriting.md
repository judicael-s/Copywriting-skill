# Career, CV, and application copywriting

Use this when writing or editing resumes, LinkedIn profiles, cover letters, application answers, portfolio pages, and recruiter-facing messages.

## Principle

The user is the product. The reader is scanning for fit, proof, and risk.

Do not write generic motivation. Build a sharp application angle first.

## Workflow

1. Identify the target role and reader.
2. Extract the role’s proof needs.
3. Select only the experiences that prove those needs.
4. Retrieve real metrics when available.
5. Decide the application angle.
6. Write scanner-friendly bullets before narrative.
7. Use the company’s semantic world lightly.
8. Close with a line that echoes the opening or role fit.

## CV bullets

Good CV bullets are compact.

They should show:

- action
- context
- method
- result or proof
- scope when useful

Pattern:

```text
[Action verb] [asset/channel/process] to [business outcome], using [method/tool], resulting in [specific proof].
```

When attribution is shared or uncertain, use:

- helped
- contributed to
- during the active audit period
- supported
- improved visibility for

Do not overclaim.

## Proof provenance

When real metrics are available, preserve:

- source path
- comparison window
- raw before value
- raw after value
- calculated growth
- attribution confidence

## Cover letters

Avoid:

- generic motivation endings
- repeating the CV in prose
- forced company metaphors
- clever concept repeated in every paragraph
- “I would be delighted to discuss” endings

Prefer:

- one strong opening image or role insight
- compact proof
- clear role fit
- final line that closes the loop

## LinkedIn positioning

Write for the scanner.

Show:

- what the person does
- for whom
- what outcome improves
- proof signals
- current direction

Avoid keyword stacking. Convert keyword lists into an action-led promise.

Weak:

```text
Content, SEO, AI, Omnichannel, Conversion.
```

Stronger:

```text
I build organic growth systems that turn search demand into pages, journeys, and offers people act on.
```

## Output blocks

When the user asks to “put it in the resume,” provide:

- summary options
- target-role experience bullets
- shorter variants
- wording bank
- avoid-list
