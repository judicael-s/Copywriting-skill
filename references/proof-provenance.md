# Proof provenance

Use this whenever copy includes metrics, case studies, claims, results, audits, or client outcomes.

## Principle

Specific proof sells. Invented proof destroys trust.

Never rely on remembered or generic metrics when real project notes, audits, GA4/GSC exports, or client docs are available.

## Required provenance fields

For each proof point, preserve:

- source path or source name
- comparison window
- raw before value
- raw after value
- calculated change
- attribution confidence
- whether the result is causal, contributed, or merely correlated

## Attribution language

Use precise verbs:

| Confidence | Wording |
|---|---|
| High causal confidence | increased, reduced, generated, recovered |
| Shared contribution | helped increase, contributed to, supported |
| Time-window correlation | during the active audit period, after the change window |
| Low confidence | observed, coincided with, directional signal |

## Never do this

- invent client names
- invent percentages
- round numbers into fake precision
- claim causality from correlation
- remove caveats when attribution is shared
- use “proven” without proof

## Copy pattern

```text
Helped improve [metric] from [before] to [after] over [window], after [action], with attribution shared across [channels/teams].
```

## Public copy rule

If client data is private, anonymize the company and preserve the lesson:

```text
A Shopify store with existing traffic was losing buyers on product pages. A clearer offer and CTA structure helped recover sales that were already close to converting.
```
